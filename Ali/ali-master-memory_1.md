[PRG:95] DECISIONS | Session 27 Apr 2026 -- Key architectural decisions. (1) Dual-persona: citizen and UKCP-affiliated are mutually exclusive sessions. One user record, two modes. MFA mandatory for affiliated. (2) Scoped roles: civic roles are 1:1 exact scope, no inheritance in either direction. Each level self-governing -- citocracy principle. (3) Role vs visibility: role governs creation/management at exact scope only. Content visibility across hierarchy is inclusion model (tab selectors), not role. Completely separate concerns. (4) Citizen user: two base states -- anon (cookie only, no record) and registered. Default posting is anonymous-traceable. Formal areas force named+logged-in. Per-post choice with settable default. (5) Auth: Supabase Auth now, Azure Entra External ID migration target. Platform-wide roles in JWT, scoped roles resolved per DB request. (6) Affiliated roles have jurisdiction (broader than single scope) -- does not contradict civic 1:1 principle as they are separate systems. (7) Proctor and Advocate: Both personas (citizen = participation role; affiliated = oversight function).
